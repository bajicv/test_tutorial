# cgMLST with chewBBACA

## 🎯 Learning goals

By the end of this practical, you should be able to:

- understand what cgMLST is used for in bacterial genomic surveillance
- run allele calling with `chewBBACA`
- identify the main output files needed for interpretation and visualization

## Overview

In this practical, we will compare **Klebsiella pneumoniae** genomes using cgMLST.

The general workflow is:

```text
FASTQ files
  ↓
Bactopia
  ↓
Assembly FASTA files
  ↓
chewBBACA allele calling → chewBBACA ComputeMSA → IQ-TREE on chewBBACA-generated MSA
  ↓
cgMLST profile
  ↓
Minimum-spanning tree
```

In the previous practical, we used Bactopia to assemble the FASTQ files. In this practical, we will use the resulting assembly FASTA files as input for chewBBACA.

## 6. Determining allelic profiles

```bash
# move to dir where you will run chewbbaca
cd ~/2026-Workshop-HSPA-Tunisia/analyses/chewbbaca/

# run chewbbaca allele calling
chewBBACA.py AlleleCall \
    -i assemblies \
    -g /shared/chewbbaca_prepared_schema \
    -o allele_call_out \
    --cpu 8 \
    --output-masked \
    --hash-profile sha256 \
    --no-inferred 
```

> [!NOTE]
> `--no-inferred` ensures that the process will not add the sequences of inferred alleles (INF) to the schema.
> Allelic profiles will still include the allele identifiers attributed to the inferred alleles.
> **Use this parameter if the schema is being accessed by multiple processes/users simultaneously**.

---

## 7. Generating an interactive report to evaluate allele calling results

```bash
chewBBACA.py AlleleCallEvaluator \
    -i allele_call_out \
    -g /shared/chewbbaca_prepared_schema \
    -o allele_call_evaluator_out \
    --cpu 8
```

> [!NOTE]
> `masked_profiles.tsv`: A TSV file containing the masked input profiles (the allelic profiles in the `results_alleles.tsv` file are masked to remove INF- prefixes and substitute all non-EXC and non-INF classification by 0).
> `cgMLST_profiles.tsv`: A TSV file that contains the allelic profiles for the set of core loci - meaning loci shared by 100% of the samples in that analysis (no missing alleles allowed "0", even if in one sample).

---

## 8. Visualization of Minimum Spanning Trees

You can upload the allelic profiles generated by chewBBACA (`allele_call_evaluator_out/masked_profiles.tsv`, or `allele_call_evaluator_out/cgMLST_profiles.tsv`) and associated metadata to software such as [PHYLOViZ Online](https://online2.phyloviz.net/index) to visualize Minimum Spanning Trees (MSTs) and perform various dataset operations that allow you to explore and analyse the results.

It is also possible to use other software such as [GrapeTree](https://github.com/achtman-lab/GrapeTree) to visualize MSTs based on the allelic profiles generated by chewBBACA, but this requires some data formatting to convert the allelic profiles generated by chewBBACA to a format compatible with GrapeTree (more information about the format supported by GrapeTree [here](https://enterobase.readthedocs.io/en/latest/grapetree/grapetree-tutorial-1-sa.html#loading-data-into-grapetree-with-the-stand-alone-version)).

## 9. Interpret the results

Try to answer:

1. How many samples were included in the cgMLST analysis?
2. Did all samples have many cgMLST loci present?
3. Which samples are closest together in the MST?
4. Are any samples separated from the others?
5. Could poor assembly quality explain missing loci?

## 10. OPTIONAL | Create a multiple sequence alignment for IQ-TREE

The allele calling result can also be converted into a multiple sequence alignment.
This alignment will be used in the next practical with IQ-TREE.

```bash
chewBBACA.py ComputeMSA \
  -i allele_call_out/results_alleles.tsv \
  -g /shared/chewbbaca_prepared_schema \
  -o msa_out \
  --dna-msa \
  --output-variable \
  --cpu 8
```

Check the output files.

```bash
ls -lh msa_out
```

For the next IQ-TREE practical, we will use `msa_out/sample_MSAs/dna/concat.fasta`.
