# Day 03 — De novo Genome Assembly with Bactopia

Day 03 focuses on bacterial genome assembly from sequencing reads. You will learn why genome assembly is needed, how paired-end Illumina reads can be reconstructed into contigs, and how assembly results can be assessed before downstream analyses.

We will introduce the basic concepts of **de novo genome assembly**, followed by practical genome reconstruction using [bactopia](https://bactopia.io/).

## 🎯 Learning goals

By the end of Day 03, you should be able to:

- explain the purpose of de novo genome assembly in bacterial genomics
- distinguish between reads, contigs, scaffolds, and assemblies
- understand how paired-end Illumina reads are used for bacterial genome reconstruction
- organize assembly input files and outputs in a reproducible project structure
- run a `bactopia` assembly workflow from paired-end FASTQ files
- understand the main files and directories generated by `bactopia`
- interpret basic assembly statistics, including number of contigs, total assembly length, N50, and GC content
- identify common signs of poor-quality assemblies
- understand why assembly quality control is important before annotation, typing, cgMLST, AMR screening, and phylogenetic analysis
